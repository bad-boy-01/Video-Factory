You are working on a Python project called NovelFactory. It is an AI pipeline
that converts a novel text file into a video. The pipeline runs on Kaggle
(free T4 GPU, 13 GB RAM, Python 3.10+).

The architecture is a compiler: text goes through planning stages that produce
intermediate JSON manifests, then rendering stages that generate images and
audio, then FFmpeg assembles the final video.

The project is structurally complete but NOT runnable. The central API class
is a facade of empty stubs — none of the real pipeline stages are ever called.
Your job is to fix this so `python main.py --project workspace compile all`
produces a real video.

FIX 1 — core/api/compiler_api.py (MOST IMPORTANT)
──────────────────────────────
The methods plan(), render(), and assemble() currently contain only
logger.info() calls. They do not call any real code.

status() returns hardcoded fake numbers.
doctor() returns hardcoded PASS for everything.
benchmark() returns made-up timings.

WHAT TO DO:

A) Add a method _load_novel_text(self) -> str that:
   - Looks for any .txt file inside self.project_dir/
   - Reads and returns its contents
   - Raises FileNotFoundError with a clear message if none found

B) Rewrite plan() to:
   1. Call _load_novel_text() to get the novel string
   2. Build a ProjectManifest with source_text=<that string> and
      novel_hash=hashlib.md5(text.encode()).hexdigest()
   3. Build a PipelineContext with project_manifest=<manifest>,
      execution_nodes=[], state={}
   4. Build a CompilerContext wrapping the PipelineContext, plus
      self.workspace, self.registry, and queue=None
   5. Instantiate LocalLLMProvider(model_id=self.config.llm_model),
      call llm.load()
   6. Build stages list in this order:
        ChunkerStage()
        StoryBibleGeneratorStage(llm)
        SceneSplitterStage(llm)
        CastPlannerStage(llm)
        ShotPlannerStage(llm)
        CameraPlannerStage(llm)
        PromptBuilderStage()
   7. Build ContractRouter(contract_map={}) — empty map, no contracts yet
   8. Run CompilerExecutor(stages=stages, contract_router=router).run(context)
   9. Call llm.unload() in a finally block
   10. Save the final context's story_bible and scene manifest to
       self.workspace.manifests_dir as story_bible.json and scene_manifest.json
   11. Return the final context

C) Rewrite render() to:
   1. Load prompt_manifest.json from self.workspace.manifests_dir
      (written by PromptBuilderStage). If missing, raise RuntimeError
      telling the user to run plan first.
   2. Instantiate DiffusersProvider(config=self.config) and call provider.load()
   3. Instantiate DiffusersCompiler()
   4. Loop over each prompt entry, call compiler.compile_plan(entry) to get
      a ProviderRequest, then provider.generate(request) to get a PIL Image
   5. Save each image to self.workspace.outputs_dir/<shot_id>.png
      Skip shots where the file already exists (crash resume)
   6. Call provider.unload() in a finally block

D) Rewrite assemble() to:
   1. Instantiate FFmpegVideoRenderer()
   2. Collect all .png files from self.workspace.outputs_dir in sorted order
   3. Collect matching .wav files from self.workspace.base_dir/audio/
   4. Build a FrameManifest from the image list
   5. Call renderer.render_video(manifest, audio_paths,
      self.workspace.outputs_dir / "final_video.mp4")

E) Rewrite doctor() to actually check the environment:
   - torch importable? torch.cuda.is_available()?
   - diffusers importable?
   - ffmpeg in PATH (subprocess.run ["ffmpeg", "-version"])?
   - workspace dirs exist and are writable?
   - Free disk > 10 GB?
   Return real PASS/FAIL/WARN for each check.

F) Rewrite status() to read real data:
   - Count .png files in outputs_dir for rendered count
   - Count entries in prompt_manifest.json for total shots
   - Read story_bible.json for scene count
   - Check GPU via torch if available

FIX 3 — plugins/ffmpeg_renderer.py
──────────────────────────────
render_video() currently hardcodes 2 seconds per frame and ignores audio.

Fix it to:
1. Accept audio_paths as a list of Path objects (already in the signature)
2. For each frame entry, check if a matching .wav exists in audio_paths
   (match by stem: shot_018.png matches shot_018.wav)
3. If a .wav exists, get its duration with:
     import wave
     with wave.open(str(wav_path), 'r') as w:
         duration = w.getnframes() / w.getframerate()
   Use that duration in the concat list. Minimum 2.0 seconds.
4. If no .wav exists, use 3.0 seconds as the default duration
5. After generating the silent video, if any audio exists, use ffmpeg
   to mix in the audio with -shortest flag
